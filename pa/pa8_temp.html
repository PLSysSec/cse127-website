The snippet above uses both functions together to generate a random integer mod 10. ## MultiplicativeCyclicGroup ``` G = MultiplicativeCyclicGroup(); q = G.order; g = G.generator; ``` A `MultiplicativeCyclicGroup` has two fields, `order`, an integer equal to the number of elements in the group, and `generator`, a group element. `MultiplicativeCyclicGroup` also overloads two operators, `*` and `**`. Specifically, `*` is overloaded for two group elements, and `**` is overloaded for a group element, and an integer mod the group order. For example, the following are valid expressions: ``` g_squared = g * g; ga = g ** Random(IntMod(q)); ``` The first line computes the product of the generator with itself, and the second computes the generator raised to a random integer mod `q`, the group order. # Appendix B: Syntax Reference ## Assignment ``` = ; valid = validate(..) ``` Assigns the expression on the right hand side to the variable on the left hand side, creating a new variable if valid is undefined, or updating valid if it has already been defined. ## If Statement ``` if { } else { } if not valid { print(“invalid”); } else { print(“valid”); } ``` Evaluates the expression after `if` and executes the first branch if it is `true`, and the second (optional) branch if it is `false`. ## Sending/Receiving ``` party alice() { bob.send(“hi”); } party bob() { print(alice.receive()); } ``` Parties are able to communicate by calling `.send(..)` and `.. = .receive()`, where `` is the name of another party defined in the same file. `send` and `receive` are blocking operations, meaning that a protocol may deadlock, for instance if alice and bob both try to send first, instead of alice sending first, and bob receiving first. ## Calling an External Function ``` prot foo; foo::bar(); ``` Protocols can be separated into files and imported with the `prot` keyword. Then you can call functions in an external protocol with the syntax `::(..);` ## Tuple Destructuring ``` party alice() { ... sig = schnorr::sign(sk, (x, y, z)); bob.send((sig, x, y, z)); } party bob() { ... (sig, x, y, z) = alice.receive(); valid = schnorr::verify(pk, (x, y, z)); } ``` Multiple values can be grouped into a tuple, which is useful if you want to send a message that has multiple values, or generate a signature over those values. In the snippet above, alice uses `(x, y, z)` to group three variables together into a tuple to be signed, and then sends `(sig, x, y, z)` to bob. Bob destructures the tuple with `(sig, x, y, z) = alice.receive();` so that each item in the tuple can be used later in the protocol. ## Defining a Function ``` def (, , ...) { } def validate(material) { return material == 123; } ``` Defines a function with named arguments that can return a value.
