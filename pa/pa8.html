There are two unique features of this language that you will need to use in this assignment: *parties* and *cryptographic primitives*. ### Parties Protocols in this language are defined in terms of *parties* which can send and receive messages. Parties look like functions, but begin with the keyword `party`. Parties defined in the same file can communicate by calling `.send(value);` and `value = other_party.receive();`. Calls to `send` and `receive` always block until the operation is completed. For instance, the previous call to `receive` will block until `` sends a message. Likewise, the call to `send` will not return until `` is executing and receives the message. ### Cryptographic primitives Several cryptographic primitives are built in to the language. Typically, they are defined at the top of the file with the keyword `global`, so that they are in scope for all functions and parties defined in the same file. For example, the starter code makes use of an `AuthenticatedCipher` object which has the following interface: ``` # instantiate a new AuthenticatedCipher C = AuthenticatedCipher(); # encrypts plaintext using key ct = C.encrypt(key, plaintext); # decrypts ciphertext using key (decrypted is equivalent to plaintext) decrypted = C.decrypt(key, ct); ``` The `key` and `plaintext` can be any value that can be serialized, which means we can pass pretty much anything as these two arguments: You can encrypt an `Integer`, a `String`, a `Tuple`, or even an `AuthenticatedCipher`. This also means that the order of arguments is important, as it is easy to accidentally encrypt the key with the message instead of the other way around (i.e., `ct = C.encrypt(msg, key);` is incorrect but may not raise an error). We have included an appendix at the end of this page that documents some of the other cryptographic primitives that appear in the starter code, as well as how to write useful language structures (like `if` statments). The snippet above uses both functions together to generate a random integer mod 10. ### MultiplicativeCyclicGroup ``` G = MultiplicativeCyclicGroup(); q = G.order; g = G.generator; ``` A `MultiplicativeCyclicGroup` has two fields, `order`, an integer equal to the number of elements in the group, and `generator`, a element of the group. (See the [Wikipedia entry](https://en.wikipedia.org/wiki/Multiplicative_group_of_integers_modulo_n) if you are curious.) `MultiplicativeCyclicGroup` also overloads two operators, `*` and `**`. Specifically, `*` is overloaded for two group elements, and `**` is overloaded for a group element, and an integer mod the group order. For example, the following are valid expressions: ``` g_squared = g * g; ga = g ** Random(IntMod(q)); ``` The first line computes the product of the generator with itself, and the second computes the generator raised to a random integer mod `q`, the group order. ## Syntax Reference ### Assignment ``` = ; valid = validate(..) ``` Assigns the expression on the right hand side to the variable on the left hand side, creating a new variable if valid is undefined, or updating valid if it has already been defined. ### If Statement ``` if { } else { } if not valid { print(“invalid”); } else { print(“valid”); } ``` Evaluates the expression after `if` and executes the first branch if it is `true`, and the second (optional) branch if it is `false`. ### Sending/Receiving ``` party alice() { bob.send(“hi”); } party bob() { print(alice.receive()); } ``` Parties are able to communicate by calling `.send(..)` and `.. = .receive()`, where `` is the name of another party defined in the same file. `send` and `receive` are blocking operations, meaning that a protocol may deadlock, for instance if alice and bob both try to send first, instead of alice sending first, and bob receiving first. ### Calling an External Function ``` prot foo; foo::bar(); ``` Protocols can be separated into files and imported with the `prot` keyword. Then you can call functions in an external protocol with the syntax `::(..);` ### Tuple Destructuring ``` party alice() { ... sig = schnorr::sign(sk, (x, y, z)); bob.send((sig, x, y, z)); } party bob() { ... (sig, x, y, z) = alice.receive(); valid = schnorr::verify(pk, (x, y, z)); } ``` Multiple values can be grouped into a tuple, which is useful if you want to send a message that has multiple values, or generate a signature over those values. In the snippet above, alice uses `(x, y, z)` to group three variables together into a tuple to be signed, and then sends `(sig, x, y, z)` to bob. Bob destructures the tuple with `(sig, x, y, z) = alice.receive();` so that each item in the tuple can be used later in the protocol. ### Defining a Function ``` def (, , ...) { } def validate(material) { return material == 123; } ``` Defines a function with named arguments that can return a value. Variables defined as `global` in the same file are in scope in ``, as well as any other functions or structs defined in that file. ### Defining a Party ``` party (, , ...) { } party alice() { bob.send(“hello”); } ``` Defines a party that can have named arguments, like a function. Parties are the entrypoints into a protocol, they cannot be called in the protocol code (e.g. `alice();`), instead they must be invoked from the command like `./cdsl party --id `. The `` is a detail that allows multiple instances of the same protocol to run simultaneously. Parties must run with the same `` to communicate. ### Structs/Methods/Impl ``` struct Rectangle { length, width, } ``` Defines a struct named `Rectangle` that has two named fields `length` and `width`. ``` x = Rectangle { length: 10, field2 : 20 }; ``` Creates an instance of the `Rectangle` struct and assigns it to `x`. Struct fields can be accessed directly (`x_len = x.length;`). ``` impl Foo { def area(self) { return self.length * self.width; } def set_len(self, len) { Self.len = len; } } ``` Structs can also define methods inside `impl` blocks, then you can call methods (`x_area = x.area();`).
